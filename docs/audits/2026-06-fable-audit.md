## 1. Executive summary

The codebase is in better shape than its documentation. The security pipeline runs in the documented cheapest-first order on every dispatch path, tier boundaries hold, all eight children apply tier-3 precision reduction, the deterministic-processing invariant holds across all eight processing modules (including five that postdate ADR 0008), and the baseline test suite is fully green (1,665 passed / 3 scipy-conditional skips / 0 failures).

The dominant failure class is **doc-truth decay across three rapid identity transitions** (v7 rename, v8 CLI hard-removal, v9 public-flip rename). Concretely:

1. **Release-record gaps**: two PyPI-published releases (9.0.1, 9.0.2) are absent from CHANGELOG.md, and an entire new child (`strong_motion`, 5 tools, shipped on PyPI) has no CHANGELOG/ROADMAP/README record.
2. **The v8.0.0 CLI hard-removal was never propagated to live artifacts**: the shipped cue card, the recipient-install-validator agent, runtime output in `setup_help` and the demo runner, and four docs/guides still instruct recipients to run removed commands.
3. **The v9.0.0 domain-agnostic flip stopped at the public surface**: seven core framework modules still title themselves "Biosensor-to-LLM Framework", and the core vault tier carries health-recipe tool names and Strava-wired defaults.
4. **Six ADRs carry stale statuses** (0022, 0023, 0038, 0040, 0042 shipped-but-Proposed; 0021 superseded-unmarked), and ADR 0014's central enforcement claim (80% coverage floor) is enforced nowhere.

No blocker-severity finding (nothing is broken on the wire for the six shipped CLI commands or the MCP tool surface). Six findings rate HIGH.
